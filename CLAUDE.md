# CLAUDE.md

Claude Code がこのリポジトリで作業する際のガイド。作業前に必ず一読すること。

## このプロジェクトの性質

実機（プロペラを持つ飛行体 Tello EDU）を制御するソフトウェアである。
バグやロジックの不備が物理的な事故につながりうるため、通常の Web アプリより
**安全性を最優先**する。

## 作業を始める前に必ず読むファイル

1. `docs/safety.md` — 安全要件とガードレール（最重要）
2. `docs/spec.md` — 何ができたら完了かの定義
3. `docs/architecture.md` — モジュール構成・状態機械・API
4. `reference/tello-sdk2.md` — コマンド文字列の一次情報

## 守ること（ガードレール）

- **安全機構を勝手に弱体化しない。** 緊急着陸、フェイルセーフ、レート制限、
  ウォッチドッグ、状態機械のガードに関わるコードを変更・削除・無効化する場合は、
  実装を確定させる前に必ず人間に確認を求めること。テストを通すためにこれらを
  バイパスする変更は禁止。
- **`takeoff` を自律的に呼ばない。** 離陸は必ず人間の明示的な操作を起点とする。
- **コマンド文字列は記憶で書かない。** `reference/tello-sdk2.md` を参照する。
  曖昧な場合は推測せず確認する。
- **実機を前提としたコードを書いても、テストはモックで行う。** 実機での飛行確認は
  人間のステップ。`TELLO_MODE=mock` で動く状態を常に維持する。
- **スコープを守る。** `README.md` のスコープ外項目は実装しない。

## 開発コマンド

```bash
uv sync                  # 依存解決
uv run pytest            # テスト実行（モックで完結すること）
uv run ruff check .      # lint
uv run ruff format .     # フォーマット
uv run uvicorn backend.app.main:app --reload   # 起動
```

## 進め方

`docs/milestones.md` のマイルストーン単位で実装する。一度に全機能を実装せず、
各マイルストーンごとに人間のレビュー・実機確認を挟む前提で、小さく区切って提案すること。
仕様に変更が生じたら、コードだけでなく `docs/spec.md`（リビングスペック）も更新する。

## コーディング規約

- Python 3.11+ / 型ヒントを付ける / 公開関数には docstring。
- 状態遷移はすべて明示的な State Machine 経由。状態を無視した直接コマンド送信をしない
  （例外: 緊急着陸はどの状態からでも受理）。
- ログは構造化し、機体に送ったコマンドと受信したステートを必ず残す。
