# 機能仕様（リビングスペック）

このドキュメントは「何ができたら完了か」を定義する基準であり、実装の進行に合わせて
更新し続ける（リビングスペック）。各機能はユーザーストーリーと受け入れ条件で記述する。

凡例: 受け入れ条件は Given / When / Then 形式。`[ ]` は未実装、`[x]` は実装・検証済み。

---

## F-01 接続確立

> オペレーターとして、ブラウザから Tello EDU に接続し、操作可能な状態にしたい。

- [ ] Given 機体が電源 ON でステーションモードで LAN に参加している
      When 「接続」ボタンを押す
      Then 機体と SDK コマンドモードを確立し、状態が `CONNECTED` になる
- [ ] Given 接続に成功した
      When 接続完了直後
      Then 電池残量・高度・温度などのステートが UI に表示され、定期更新される
- [ ] Given 接続に失敗した（タイムアウト / IP 不達）
      When 一定時間応答がない
      Then エラー理由を UI に表示し、状態は `DISCONNECTED` のまま

## F-02 手動飛行操作

> オペレーターとして、カメラ映像を見ながら手動で機体を操作し、追従対象が映る位置まで
> 移動させたい。

- [ ] Given 状態が `CONNECTED` 以降である
      When 「離陸」ボタンを押す（人間の明示操作）
      Then 機体が離陸し、状態が `MANUAL` になる
- [ ] Given 状態が `MANUAL` である
      When 方向ボタン / キー（前後・左右・上下・旋回）を操作する
      Then 対応する移動コマンドが機体に送られ、映像に反映される
- [ ] Given 操作入力がない
      Then 機体はその場でホバリングを維持する
- [ ] 操作系は画面のオンスクリーンボタンとキーボードの両方で行える

## F-03 カメラ映像取得・表示

> オペレーターとして、機体のカメラ映像をブラウザでリアルタイムに見たい。

- [ ] Given 接続済みである
      When 映像ストリームを開始する
      Then ブラウザの映像エリアに低遅延で映像が表示される
- [ ] Given 映像が表示されている
      Then 現在の状態（MANUAL / TRACKING 等）と電池残量が映像上に重畳表示される
- [ ] Given 映像ストリームが途切れた
      Then UI に明示し、追従中であれば安全側（F-06 参照）に倒す

## F-04 追従対象の指定

> オペレーターとして、映像内で追従したいオブジェクトを矩形で囲んで指定したい。

- [ ] Given 状態が `MANUAL` で対象が映像中心付近に映っている
      When 映像上でドラッグして矩形を描く
      Then その矩形でトラッカーが初期化され、状態が `TARGET_SELECTED` になる
- [ ] Given 対象を指定した
      Then 指定矩形が映像上にオーバーレイ表示される
- [ ] Given 指定をやり直したい
      When 再度ドラッグする / 「対象クリア」を押す
      Then 対象が再設定 / 解除される

## F-05 自動追従の開始・停止

> オペレーターとして、指定した対象を機体が自動で追従する状態を、明示的に開始・停止したい。

- [ ] Given 状態が `TARGET_SELECTED` である
      When 「追従開始」ボタンを押す
      Then 状態が `TRACKING` になり、対象が映像中心に来るよう速度コマンドが送られる
- [ ] Given 状態が `TRACKING` である
      When 対象が画面内で移動する
      Then 中心とのズレに応じて滑らかに（急激でなく）追従する
- [ ] Given 追従中に対象を見失った（トラッカー失敗 / 画面外）
      Then 直ちに前進・旋回を止めてホバリングし、状態を `TARGET_SELECTED` に戻して UI に通知する
- [ ] Given 状態が `TRACKING` である
      When 「追従停止」ボタンを押す
      Then 速度コマンドを止めてホバリングし、状態が `MANUAL` に戻る

## F-06 緊急着陸・安全停止

> オペレーターとして、いつでも・どの状態からでもワンアクションで機体を安全に着陸させたい。

- [ ] Given 飛行中（MANUAL / TARGET_SELECTED / TRACKING のいずれでも）
      When 「緊急着陸」ボタンを押す / 割り当てキー（例: スペース）を押す
      Then 他のどの処理よりも優先して着陸コマンドを送り、状態を `LANDING` にする
- [ ] 緊急着陸ボタンは飛行中、常に画面上に表示され押下可能であること
- [ ] Given 通信断・ブラウザクラッシュ・映像断などの異常
      Then ウォッチドッグが検知し、自動でホバリング→着陸の安全側処理を行う
      （詳細は `docs/safety.md`）

## F-07 切断

> オペレーターとして、操作を終えたら機体との接続を安全に解除したい。

- [ ] Given 機体が着地している（飛行していない）
      When 「切断」ボタンを押す
      Then 映像ストリームとコマンドモードを終了し、状態が `DISCONNECTED` になる
- [ ] Given 機体が飛行中である
      When 「切断」を押す
      Then いきなり通信を切らず、**自動的に着陸コマンドを送り、着地を確認してから切断する**
      （飛行中の即時切断は禁止。safety S-7）

---

## 非機能要件（抜粋）

- モックモード（実機なし）で F-01〜F-07 の UI とロジックが一通り動作すること。
- 機体に送った全コマンドと受信ステートが構造化ログに残ること。
- 想定ユーザーは単一オペレーター・ローカル LAN 内利用。認証は設けない。
