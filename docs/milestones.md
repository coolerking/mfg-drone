# 開発マイルストーン

一括で全機能を実装せず、小さく検証可能な単位に分割する。各マイルストーンは
モックモードでの検証を完了の前提とし、実機確認は人間が立ち会う独立ステップとする。

各マイルストーンは GitHub Issue として切り出してもよい。

---

## M0 足場づくり

- [ ] `uv sync` で環境が作れる（`pyproject.toml` 確定）
- [ ] FastAPI が起動し、`frontend/index.html` が配信される
- [ ] `TelloController` の抽象インターフェース（`drone/base.py`）と
      モック実装（`drone/mock.py`）のスケルトン
- [ ] 状態機械（`state_machine.py`）の骨格と単体テスト
- [ ] 構造化ログの基盤

**完了判定:** モックモードでサーバが起動し、空 UI が表示され、状態機械のテストが通る。

## M1 接続 / 切断（F-01, F-07）

- [ ] `/api/connect` `/api/disconnect` と状態遷移
- [ ] `/ws/state` で電池残量などを定期 push
- [ ] UI に接続状態・電池残量パネル
- [ ] 飛行中の切断禁止ガード（S-7）

**完了判定（モック）:** 接続→ステート表示→切断が UI から行える。
**実機確認:** 実機 IP で接続でき、電池残量が取得できる。

## M2 手動操作 + 映像表示（F-02, F-03）

- [ ] `/api/takeoff` `/api/land` `/api/move`
- [ ] `/ws/video` での映像配信、UI の映像エリア表示
- [ ] オンスクリーンボタン + キーボード操作
- [ ] 速度・移動量のクランプ（S-3）
- [ ] 映像断の検知（S-4 の一部）

**完了判定（モック）:** ダミー/Webカメラ映像が表示され、操作で状態とログが動く。
**実機確認:** 離陸→手動移動→着陸が安全に行える（広い屋内・プロペラガード）。

## M3 緊急着陸とウォッチドッグ（F-06）

> M2 と並行 or 直後。実機を本格的に飛ばす前に S-1/S-4 を必ず入れる。

- [ ] `/api/emergency`（最優先・常時受理。`land` ベース）
- [ ] 緊急着陸ボタン常時表示 + キー割り当て（例: スペース）
- [ ] WebSocket heartbeat 途絶検知 → ホバリング → 自動着陸
- [ ] 映像断・例外時のフェイルセーフ
- [ ] `tests/test_safety.py` で各フェイルセーフを検証

**完了判定（モック）:** 各種異常を注入すると安全側に倒れることがテストで確認できる。
**実機確認:** 飛行中に緊急着陸ボタン / ブラウザ切断で安全に降りる。

## M4 追従対象の指定（F-04）

- [ ] 映像上のドラッグで矩形選択 → `/api/target`
- [ ] OpenCV トラッカー（CSRT）初期化、矩形オーバーレイ
- [ ] `/api/target/clear`、状態 `TARGET_SELECTED`

**完了判定（モック）:** 録画/Webカメラ映像で対象を指定し、トラッカーが追跡を継続する。

## M5 自動追従（F-05）

- [ ] `/api/track/start` `/api/track/stop`
- [ ] 中心ズレ → `rc` への制御則（出力上限つき）
- [ ] 対象ロスト時のホバリング・フォールバック（S-5）
- [ ] 追従ループのレート制限（S-2）

**完了判定（モック）:** 仮想/実映像で対象が動くと制御出力が妥当に変化し、ロスト時に停止する。
**実機確認:** 物（箱・マーカー等）を対象に、低速で安定して追従する。人・顔は対象にしない。

## M6 仕上げ

- [ ] 全機能の通し動作確認（モック → 実機）
- [ ] `docs/spec.md` の受け入れ条件をすべて `[x]` にできる状態
- [ ] README のセットアップ手順どおりに第三者が起動できる
- [ ] ログ・エラー表示の整備

---

## 進め方のメモ

- 安全機構（M3）は「最後にまとめて」ではなく、実機を飛ばし始める M2 と同時期に入れる。
- 仕様変更が出たら、コードと同時に `docs/spec.md`（リビングスペック）を更新する。
- 各マイルストーン完了時に人間がレビューし、実機確認を挟んでから次へ進む。
